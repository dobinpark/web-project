# 매일데이(Maeil Day)

직장인들의 새로운 활력소,
이곳이 바로 매일데이 커뮤니티입니다.

# 💡 Topic
* **직장인 소통 커뮤니티 플랫폼**
* 직장인들을 대상으로 지루한 일상속에 서로 소통하며 공감대를 형성할 수 있는 소통의 장으로 만든 서비스
* 직장인들의 창의성을 이용하여, 매일데이에서 직접 상품을 등록, 판매할 수 있는 서비스
* (예정) 직장인들이 속해있는 회사 근처에 맛집 지도와 맛집 리스트를 확인할 수 있는 서비스

# 📝 Summary
대학생들의 커뮤니티 서비스인 에브리타임을 보면서 ‘직장인들을 위한 에브리타임은 없을까?’라는 생각을 했습니다.

에브리타임처럼 깔끔하고 직관적인 커뮤니티를 직장인들에게 제공한다면, 보다 편하게 서로 소통하면서
매일데이라는 커뮤니티 서비스를 일상 속에서 언제든지 이용할 수 있도록 만든 서비스입니다.

# ⭐️ Key Function
* 매일게시판
  * 현재 자유게시판, 질문게시판이 존재하지만 추후에 하나의 카테고리를 만들어서 다양한 게시판을 만들 계획
  
* 매일몰
  * 직장인들의 창의성을 발휘하여 매일몰에 상품을 등록, 판매할 수 있음
  
* 매일 맛집 (예정)
  * 회사 근처에 맛집 지도와 맛집 리스트를 제공하여 점심 고르는 시간을 단축할 수 있는 편의성을 제공할 수 있음

# 🛠 Tech Stack
Java , Spring , HTML/CSS , Thymeleaf , Bootstrap , Gradle , MySQL , Querydsl

# ⚙️ Architecture
MVC

# 🧑🏻‍💻 Team
백엔드 개발자 1명

# 🤚🏻 Part
**전체 기획 및 개발**

# 🤔 Learned
* 첫 웹 애플리케이션 프로젝트를 제작하면서, 생각보다 많은 비즈니스 로직과 라이브러리가 필요하다는 것을 알게 되었음.
* 많은 데이터를 다루게 되다 보니 DB 설계의 중요성을 깨닫게 됨.
* 추후에 서비스를 확장할 예정인데 단순히 기능 구현에 초점이 아닌, 유지보수를 위해서 리팩토링과정을 통해 불필요한 코드를 개선하고 보다 효율적인 클린 코드로 웹을 개발해야겠다는 생각이 듦.
* 이전의 프로젝트에서는 순수 JPA를 사용했었다면, 이번 프로젝트에서는 Spring Data JPA를 사용법을 알게 됨.
* AWS 배포를 목적으로 AWS 공부를 하고, 배포를 위해 보안 부분에 공부해볼 수 있었음.
* 프로젝트를 진행하면서 많은 부분에서 부족함을 느끼고 있기 때문에 이 프로젝트의 완성을 위해 공부계획을 세울 수 있었음.
